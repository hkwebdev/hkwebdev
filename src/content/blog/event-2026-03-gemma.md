*Google AI: Gemma NPCs + The Browser is the Brain: Running Google Gemma Locally with WebGPU*

In this session of HKWD we dive into the world of local models with Google AI. The evening will be held at Garage Society with their support along with AngelFlow. The night is split into two parts. One on video game development of Non-Player Characters using GenAI and the other will be The Browser is the Brain: Running Google Gemma Locally with WebGPU.

## Talk 1: Video game development of Non Player Characters and using GenAI

Using AI in video games is not a new idea and has been used a long time for things like game mechanics, character movements, path finding, dynamic levels and maps. However using it for non-playable characters is new and Google Gemma is designed exactly for that. We take a look at how to get up and running with Gemma in Unity and some of the alternatives. 

**Gemma**
Google has a great library designed just for Non-Player Characters and we will explore how to use it in Unity: [https://ai.google.dev/gdc](https://ai.google.dev/gdc)

## Talk 2: The Browser is the Brain: Running Google Gemma Locally with WebGPU

While many associate AI with heavy cloud costs and latency, Google’s Gemma is small and efficient enough to run directly in the user's browser. In this session, Frankie (GDG Hong Kong) explores the frontier of "Local AI" for the web. We will discuss how developers can use Google’s tools to run Gemma models on-device via WebGPU. This approach eliminates API costs and privacy concerns, allowing web developers to create highly responsive, AI-powered interfaces that feel as fluid as a modern video game—all without a backend.
